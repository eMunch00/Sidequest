# :bookmark: Markdown Language Syntax:
![MD_Header](./MD.jpg
)
* *The following are symbols used in Markdown Language formatting.*
## HEADERS & ORGANIZATION:
* `#` ... *Header-Main:* Top Level Project Title
* `##` ... *Header-Section Title:* Use as Section Titles
* `###` , `####` ... *Header-Sub-Sections:* Up to 6 header subs available
* `---` or `___` ... *Separation-Line:* creates visible a line across page... *Example Below*
--- 
## LISTS:
* `-` ... *Bullet-Point:* Use for unordered lists
* `*` ... *Bullet-Point:* use for unordered lists
* `Tab *` ... *Nested-Bullet-Point:* Create an unordered list within another list.
* `1.` ... *Numerated-List:* entering `1.` at each line will format a numbered list.
## TASK-LIST / CHECK-BOXES:
* `[]` ... *Check-Box:* Empty brackets will be displayed as a check-box []
* `[x]` ... *Checked-Box:* will display as a checked box [x]
## TEXT FORMAT:
* `*[...]*` ... *Italics:*
* `**[...]**` ... *Bold:*
* `~~[...]~~ ` ... *Strike-Trough:*
* `\[...]\` ... *Ignore Syntax:* Use to display characters that MD uses in formatting.
## CODE BLOCKS:
* \`[...]`\ ... *Basic Code Block:* use to display text as a small code block
* \```[...]```\ ... *Extended Code block:* Use to display several lines of code as a text block, will provide a copy icon
## LINKS & IMAGES:
* `[...](...)` ... *Link:*
    * `[Display_Text]` ... Text in brackets will be displayed as link.
    * `(Link_Path)` ... enter Link Path in parentheses.
        * **Example 1:** `[Link_To_Website](https://website.com)`
        * looks like this: [Link_To_Website](https://google.com) and directs browser to 'website.com'
        * **Example 2:** `[Link_To_Document](./folder_path2)`
        * looks like this: [Link_To_Document](./README.md)
        * *Note* `./` relevant path from current directory to sub-directory.
        * *Or* `../` relevant path from current directory to parent directory.
* `![...](...)` ... *LInk Image:* Image file must be located in file directory. 
    * `![Image_Title]` ... Text in brackets will be used as image name.
    * `(File_Path)` ... enter path to image file in parentheses.
## FOOTNOTES:
* `...[^1]` ... *Foot-Note @ Text:* add after text needing citation
* `[^1]...` ... *Foot-Note @ Footer:* add to bottom of document for citation reference.
## GITHUB - MENTIONS & ISSUES:
* `@username` ... *Mention:* notifies user mentioned
* `#HASH` ... *Issue or Pull-Request ID:* enter a pull request# or issue# and github will provide a link to its location.
